# 🦠 Plankton — 进化之旅

一款基于 Web 的深海微生物进化生存游戏。从一粒光子开始，吞噬粒子、捕食低级生物，经历 8 个物种形态的进化，最终成为吞噬一切的利维坦。

**[在线体验](https://yophelo.github.io/plankton/)** · **[Netlify 镜像](https://plankton-game.netlify.app/)**

## 游戏玩法

- 玩家控制一个微生物在 4000×4000 的深海世界中生存
- 吸收散布在世界中的能量粒子来积累进化能量
- 捕食比自己等级低的 AI 生物可以快速获取能量
- 被比自己等级高的生物碰到则会被消灭
- 同等级生物之间不会互相伤害
- 完成 5 个进化阶段后升级为下一物种
- 从 I. 光子 进化到 VIII. 利维坦 即为胜利

### 得分系统

| 项目 | 分值 |
|------|------|
| 最高等级 | 等级 × 1000 |
| 存活时间 | 秒数 × 10 |
| 击杀数 | 每次 × 100 |
| 胜利奖励 | 10000 |

## 物种系统

游戏包含 8 个进化阶段，每个物种有独特的视觉外观和行为特征：

| 等级 | 名称 | 描述 | 基础速度 | 狩猎速度 |
|------|------|------|----------|----------|
| I | 光子 | 微尘级 - 神经质的高频移动 | 0.035 | 0.04 |
| II | 飞梭 | 幼体级 - 僵硬的冲刺与滑行 | 0.028 | 0.04 |
| III | 水母 | 腔肠级 - 径向收缩与喷射 | 0.025 | 0.032 |
| IV | 蛇刺 | 环节级 - 正弦波游动与倒刺 | 0.021 | 0.032 |
| V | 双螯 | 甲壳级 - 重装甲与机械剪切 | 0.014 | 0.032 |
| VI | 滑翔者 | 蝠鲼级 - 优雅的波浪飞行 | 0.013 | 0.035 |
| VII | 九头蛇 | 聚合级 - 多核心独立索敌 | 0.011 | 0.035 |
| VIII | 利维坦 | 巨兽级 - 吞噬一切的引力场 | 0.011 | 0.032 |

### 物种视觉特征

- **光子**: 单点发光体，高频抖动拖尾
- **飞梭**: 短链体，青色发光
- **水母**: 径向触手结构，红色脉冲
- **蛇刺**: 长链正弦波动，绿色倒刺装饰
- **双螯**: 双臂钳形结构，橙色机械感
- **滑翔者**: 宽翼展波浪运动，银白色
- **九头蛇**: 多头分支结构，紫色独立运动
- **利维坦**: 超长链体，齿轮装饰，黑洞吞噬特效

## 进化机制

每个物种需要完成 **5 个进化阶段** 才能升级到下一物种：

| 阶段 | 效果 |
|------|------|
| 阶段 1 | 生成第 2 个同类生物（群体扩张） |
| 阶段 2 | 生成第 3 个同类生物（群体扩张） |
| 阶段 3 | 升级复杂度 1（视觉/结构增强） |
| 阶段 4 | 升级复杂度 2（视觉/结构增强） |
| 阶段 5 | 进化为下一物种 |

### 能量需求

每个阶段需要的能量 = `40 × 2^level`

| 等级 | 每阶段能量 |
|------|-----------|
| 0 (光子) | 40 |
| 1 (飞梭) | 80 |
| 2 (水母) | 160 |
| 3 (蛇刺) | 320 |
| 4 (双螯) | 640 |
| 5 (滑翔者) | 1280 |
| 6 (九头蛇) | 2560 |
| 7 (利维坦) | 5120 |

## 控制方式

游戏支持两种控制模式，在开始界面选择：

### 🕹️ 虚拟摇杆（默认，推荐手机使用）

- 手指按下屏幕任意位置作为摇杆中心
- 拖动手指控制移动方向和速度
- 松手后角色停止移动
- 摇杆有半透明视觉反馈

### 🎯 触摸跟随

- 角色朝触摸/鼠标位置移动
- 适合 PC 鼠标操作
- 松手后角色保持最后方向漂移

## 模块详解

### `config.js` — 全局配置

核心常量定义：

```javascript
WORLD_SIZE = 4000        // 世界尺寸（-2000 到 +2000）
PARTICLE_COUNT = 4000    // 场景中维持的粒子数量
BASE_STAGE_ENERGY = 40   // 基础进化能量
INITIAL_AI_COUNTS = {    // 各等级 AI 初始数量
  0: 18, 1: 12, 2: 8, 3: 5, 4: 3, 5: 2, 6: 1, 7: 1
}
SPECIES = [...]          // 8 个物种的完整配置
```

### `game.js` — 游戏主循环

职责：
- 管理游戏状态（开始、运行、结束、胜利）
- 每帧执行 `update()` → `draw()` 循环
- 协调所有子系统（AI、粒子、相机、碰撞）
- 处理进化逻辑和得分计算
- 绘制世界边界光墙
- 屏幕震动效果

关键方法：
- `init()` — 初始化/重置游戏状态
- `update()` — 物理更新、碰撞检测、AI 行为
- `draw()` — 渲染所有可见元素
- `advanceStage()` — 处理进化阶段推进
- `drawBoundary()` — 绘制动态边界光墙
- `spawnAICreature()` — AI 生成（含安全区避让）

### `camera.js` — 相机系统

- 平滑跟随玩家群体中心
- 根据进化等级自动调整缩放（高级生物视野更大）
- 提供世界坐标 ↔ 屏幕坐标转换
- 视口裁剪计算

缩放级别：`[1.6, 1.4, 1.2, 1.0, 0.85, 0.7, 0.6, 0.5]`

### `input.js` — 输入系统

双模式设计：
- **TOUCH_FOLLOW**: 传统触摸/鼠标跟随
- **JOYSTICK**: 虚拟摇杆（任意位置按下即为中心）

摇杆参数：
- 半径 50px，外圈 70px
- 输出归一化方向向量 (-1 到 1)
- 支持多点触控（只追踪第一个触点）
- PC 端鼠标按住拖动模拟摇杆

### `particles.js` — 粒子系统

- 维持场景中 4000 个能量粒子
- 粒子有微小随机漂移速度
- 视口外粒子自动回收，视口附近补充新粒子
- 被吃掉的粒子通过索引移除
- 生物死亡时爆散为粒子回归场景

### `ai.js` — AI 系统

**群集行为 (Flocking)**：
- 凝聚力 (Cohesion) — 同组生物聚集
- 分离力 (Separation) — 避免过于拥挤
- 逃避 (Flee) — 远离高等级捕食者
- 追猎 (Prey pursuit) — 狩猎模式下追逐低等级猎物
- 边界回避 — 靠近世界边缘时转向

**AI 管理器 (AIManager)**：
- 管理 AI 群组（每组有独立能量和进化状态）
- AI 也会进化（吃粒子积累能量 → 升级）
- 被击杀的 AI 在 600 帧后重生
- AI 群组进化路径与玩家相同（扩张 → 复杂度 → 升级）

### `audio.js` — 音频系统

完全使用 Web Audio API 程序化生成，零外部文件：

**BGM（深海环境音）**：
- 40Hz 正弦波低频 drone + LFO 调制
- 双锯齿波 pad 经低通滤波器缓慢扫频
- 800Hz 高频微光点缀 + 振幅 LFO

**音效**：
| 事件 | 音效描述 |
|------|----------|
| 吃粒子 | 高频正弦 blip（600-1200Hz 上扫，0.1s） |
| 击杀 | 带通噪声爆发 + 低频正弦下降（150→40Hz） |
| 阶段进化 | 四音上升琶音（400-800Hz） |
| 物种升级 | 锯齿波大扫频（200→1200Hz）+ 闪光音 |
| 死亡 | 锯齿波下降（400→60Hz）+ 噪声衰减 |
| 狩猎激活 | 方波上升（100→200Hz，0.2s） |

### `chain.js` — 链式骨骼系统

所有生物的物理结构基础：
- `Chain(count, distance, startRadius, endRadius, stiffness)`
- 头部跟随目标位置，后续节点通过刚度约束跟随
- 每个节点有独立半径（用于碰撞检测和渲染）
- 支持最大距离约束防止过度拉伸

### `creatures/base.js` — 生物基类

所有 8 种生物的公共逻辑：
- 状态管理（正常、狩猎、冷却、攻击、死亡）
- 粒子碰撞检测（链节点 + 中心碰撞体）
- 生物间碰撞检测（中心对中心、中心对链、链对链）
- 胜负判定（高等级吃低等级）
- 死亡粒子爆散
- 狩猎计时器和冷却
- 复杂度升级接口

## 本地开发

### 环境要求

- Node.js 18+
- npm 9+

### 安装与运行

```bash
# 克隆仓库
git clone https://github.com/yophelo/plankton.git
cd plankton

# 安装依赖
npm install

# 启动开发服务器（热重载）
npm run dev

# 构建生产版本
npm run build

# 预览生产构建
npm run preview
```

开发服务器默认运行在 `http://localhost:5173`。

### 手机调试

1. 确保手机和电脑在同一局域网
2. `npm run dev -- --host` 启动（暴露局域网地址）
3. 手机浏览器访问显示的局域网 IP 地址
4. 横屏使用

## 构建与部署

### GitHub Pages（自动）

推送到 `main` 分支自动触发 GitHub Actions：
1. 安装依赖 (`npm ci`)
2. 构建 (`vite build --base=./`)
3. 部署到 GitHub Pages

**首次设置**：仓库 Settings → Pages → Source 选择 "GitHub Actions"。

### Netlify（自动）

已配置 `netlify.toml`：
- 构建命令：`npm run build`
- 发布目录：`dist/`
- 推送到 `main` 自动触发部署

**首次设置**：在 Netlify 关联 GitHub 仓库即可。

### 手动部署

```bash
npm run build
# dist/ 目录即为可部署的静态文件
```

## 性能优化

### 渲染优化

- **视口裁剪**: 只渲染相机可见范围内的生物和粒子
- **粒子回收**: 远离视口的粒子自动回收，附近补充
- **碰撞检测预筛选**: 先做距离平方检查，再做精确碰撞
- **链节点采样**: 生物间碰撞不检查每个节点，按步长采样

### 移动端优化

- 世界大小 4000（而非 10000），减少空旷区域
- 粒子数量 4000，平衡视觉效果和性能
- Canvas 2D 而非 WebGL，兼容性更好
- 无外部资源加载（音频程序化生成）
- 触摸事件 `passive: false` 防止滚动干扰

### 内存管理

- 粒子数组动态维护固定大小
- 死亡生物立即从数组移除
- AI 重生使用定时器而非保留引用

## 设计决策

### 为什么世界大小是 4000？

原始设计为 10000×10000，针对 1920×1080 PC 屏幕。手机横屏约 800×360 CSS 像素，zoom 1.6 时可视范围仅 500×225 世界单位。在 10000 的世界中，玩家视野只覆盖 AI 分布区域的 0.45%，几乎遇不到敌人。缩小到 4000 后密度提升约 6 倍，体验流畅。

### 为什么用虚拟摇杆？

触摸跟随模式下，手指必须放在目标位置，会遮挡视野。摇杆模式下手指可以放在任意位置，通过相对位移控制方向，不遮挡游戏画面，更适合手机操作。

### 为什么用程序化音频？

- 零网络请求，无加载延迟
- 包体积极小（纯代码）
- 可以根据游戏状态动态调整参数
- 深海主题适合合成器风格的暗色调音效

### 为什么有出生安全区？

世界缩小后 AI 密度增加，玩家出生在 (0,0) 中心，低级 AI 也集中在中心区域。没有安全区的话，玩家一出生就可能与 AI 重叠导致立即死亡。400 单位的安全区给玩家约 2-3 秒的反应时间。

### 为什么边界有弹性回弹？

硬停边界在手机上体验差——手指还在滑动但角色不动，玩家不知道发生了什么。弹性回弹 + 屏幕震动 + 视觉光墙三重反馈，让玩家明确感知到"这里是边界"。
